핵심 결정 · API vs UI

진짜 저장 버튼을 누른다

방식무결성인증치명적 단점
API 직접 호출위험토큰 책임검증·워크플로 건너뜀 · 알림·결재·채번 누락
API 콜 재현시퀀스 누락—단일 콜만 재현 시 상태 깨짐 · 암호화 페이로드
UI 조작(버튼 클릭) ★안전세션 상속DOM 취약 → 채움 난이도로 이동
저장을 클릭하면 페이지가 내부 호출·알림·결재·채번을 정상적으로 수행합니다. 반면 API를 직접 호출하면 그 중간 이벤트가 누락되는데, 이것이 API 방식을 버린 결정적 이유입니다.
안전의 기준 · 쉬운 말로

다시 하면 되는 것과 한 번 하면 끝인 것

↩️ 다시 하면 되는 행동

보기·이동·검색·입력폼 채우기(저장 전). 틀려도 다시 하면 끝 → 마음껏 자동.

⚠️ 한 번 하면 끝인 행동

저장·삭제. 누르면 못 되돌림 → 신중(확인 한 번·금액 상한).

에이전트는 행동을 하기 전에 매번 "물어볼지, 그냥 할지"를 정하는데, 이때 위험등급·신뢰도·모드를 함께 봅니다. 사용자를 답답하게 하는 건 마지막 저장이 아니라 중간의 잦은 확인들인데, 에이전트가 그래프 DB에 경험을 쌓을수록 이런 확인이 줄어듭니다.

역할 분담

정하는 건 에이전트, 하는 건 익스텐션

🧩 익스텐션 (손·눈)

DataExtractor·FormExtractor 화면·폼 인식 · ActionExecutor 실제 폼 입력·클릭 · GovernancePlane 마스킹·감사

🧠 에이전트 (계획)

IntentParser·Planner·Resolver 계획·정식 코드 찾기 · Orchestrator·Gate·Verifier 진행·결정·검증 · MemoryWriter

핵심은 fill-plan입니다. "어느 칸에 무슨 값"을 검사할 수 있는 계획으로 먼저 만들기 때문에, 안전·HITL·테스트가 한 번에 해결됩니다.
한 작업이 흐르는 법 · 등록 예시

이동 → 인식 → 정식 코드 찾기 → 채움 → 확인 → 저장 → 검증

①

이동

그래프 경로로 목적 화면 도달(딥링크 / 메뉴 / 사람).

②

인식

FormContext 추출 — 품목 검색위젯·수량·라인그리드·저장버튼.

③

정식 코드 찾기

"볼펜" → 정식 품목코드(검색위젯). 모호하면 HITL로 고르기.

④

채움

안전 주입으로 라인 입력(controlled input 대응).

⑤

확인

fill-plan 미리보기 → 사용자 승인(되돌릴 수 없는 직전).

⑥

저장

클릭 — 페이지가 알림·결재·채번을 정상 수행.

⑦

검증

결과(레코드ID/에러) 확인 → 성공을 메모리에 적재.

등록만이 아니다

조회·수정·등록·삭제 = 같은 루프

요청한 번 하면 끝(저장)도입 순서 · 이유
조회없음1순위 — 가장 안전·빈번, v0.2 읽기 재활용
수정1번2순위 — 기존 레코드 기반, diff 확인이 자연스러움
등록1번3순위
삭제·승인매우 신중4순위 — 가장 높은 게이트
요청 종류마다 따로 프로그램을 만들지 않습니다. 에이전트가 같은 기본 동작(이동·검색·채우기·클릭)을 요청에 맞게 조합할 뿐입니다. 그래서 저장이 없는 조회는 끝까지 자동으로 흐르고, 저장이 있는 수정·등록·삭제에서만 마지막에 "이대로 저장할까요?" 확인이 들어갑니다.
어떻게 화면에 가나

길은 주되, 상태를 보며 간다

1️⃣ 딥링크

있으면 → 그 화면으로 직행. 가장 빠르고 튼튼.

2️⃣ 메뉴 자동 조작

딥링크 없으면(웹스퀘어) 그래프의 클릭 경로(대>중>소)를 따라감.

3️⃣ 사람 핸드오프

막히면 "그 화면으로 가주세요" → 도착 감지 후 이어받기.

에이전트가 목적 화면으로 직접 이동하고 그 과정을 사용자가 지켜봅니다. 모호하거나 되돌릴 수 없는 지점에서는 사용자가 직접 확인합니다.
우리의 무기

모델에 없는 사내 사전 지식을 그래프로

● 노드

화면·상태(목록·상세·폼).

→ 엣지

이동 = 클릭 경로(딥링크 없으니 URL 아닌 시퀀스).

▤ 폼 스키마

필드·필수·옵션·스텝 → fill-plan에 사용.

클로드 인 크롬은 GitHub를 학습으로 이미 알지만, 우리 사내 시스템은 모릅니다. 그 빈자리를 우리 그래프가 채웁니다. 그래서 이동은 헤매는 탐색이 아니라 그래프 경로 탐색이 됩니다.
변하는 화면 따라잡기

능동 부트스트랩 + 수동 유지

① 능동 (도입 시 1회)

메뉴·목록·빈 폼까지 이동만 하며 그래프 생성. 빠른 커버리지.

② 수동 (실사용 중 상시)

라이브 화면 vs 저장 그래프 diff → 자동 갱신/추가. 쓰면서 자란다.

능동 크롤은 "이동만" 합니다. 저장·삭제·승인처럼 데이터를 바꾸는 버튼은 절대 누르지 않습니다(실제 거래가 발생하니까요). 이를 위해 금지 버튼 목록·읽기 전용 권한·관리자 감독을 둡니다.

🛡️ 거버넌스

세션 권한 상속(새 인증 0) · 마스킹 · 미리보기 후 사람이 저장 · 감사로그 · 인젝션 가드.

🪜 단계 (위험 낮은 데서)

데모폼 PoC → 조회 → 수정 → 등록 → 삭제 → 지식그래프 가동.

클로드 인 크롬을 따라 하는 데는 힘을 적게 쓰고, 우리만의 무기(지식 그래프·도메인 최적화·거버넌스)에 힘을 집중합니다.
