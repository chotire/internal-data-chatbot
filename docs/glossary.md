# 용어집 (Glossary)

김반장(金班長) 프로젝트 대화·문서에 나오는 기술 용어를 *쉬운 말*로 정리. 각 항목 = 한 줄 정의 + "왜 우리에게 중요한가".

> 김반장 = 이 챗봇/에이전트의 별칭. v0.2(화면 읽기) → v0.3(툴) → v0.4(행동 자동화) → v0.5(사내지식) → v0.6(자기개선)로 발전. 상세는 [docs/architecture/](architecture/).

---

## 1. 익스텐션 · 브라우저

- **익스텐션(Extension)** — 크롬/엣지에 설치되는 부가 프로그램. 김반장의 *채널*(화면을 읽고·조작하는 손과 눈)이 여기 산다.
- **MV3 (Manifest V3)** — 크롬 익스텐션의 현재 표준 규격. service worker·content script·side panel·권한 모델을 정의. *우리 익스텐션이 따르는 규격.*
- **Service Worker (MV3 백그라운드)** — 익스텐션의 백그라운드 두뇌. **수명이 짧아 쉬고 있으면 크롬이 꺼버린다.** → *긴 다단계 에이전트 루프엔 골치*(꺼지면 작업 끊김). offscreen document·keep-alive·상태를 storage에 저장해 대응.
- **Content Script** — 웹페이지 안에 주입돼 DOM을 읽고 조작하는 코드.
  - **ISOLATED world** — 기본 실행 공간. 페이지의 전역 변수(React/WebSquare 객체)는 *안 보임*.
  - **MAIN world** — 페이지와 같은 공간. 전역 데이터(가상스크롤 전체·차트)를 읽으려면 여기에 주입해야 함. (v0.2의 데이터레이어 읽기)
- **Side Panel** — 브라우저 오른쪽에 붙는 패널. 김반장 챗 UI가 여기 뜸.
- **host_permissions** — 익스텐션이 어떤 도메인에서 동작할지 허용 목록.
- **`chrome.scripting` / `captureVisibleTab`** — 코드 주입 API / 화면 스크린샷 API(비전용).
- **CDP (Chrome DevTools Protocol)** — 크롬을 외부에서 원격 조종하는 저수준 프로토콜. Playwright가 이걸로 브라우저를 몬다.
- **Deep Link(딥링크)** — 특정 화면으로 바로 가는 URL/주소. 있으면 메뉴 안 타고 직행. **웹스퀘어 등 레거시는 딥링크가 없을 때가 많다.**
- **SPA (Single Page App)** — 페이지 새로고침 없이 화면만 바뀌는 웹앱. URL이 안 변해서 "지금 어느 화면인지" 판별이 까다로움.

## 2. 인식 방식 (화면을 어떻게 이해하나)

- **DOM (Document Object Model)** — 웹페이지의 구조(요소 트리). *결정론적·저비용·테스트 가능.* 단 canvas·비시맨틱 화면은 잘 안 보임.
- **a11y / 접근성 트리(Accessibility Tree)** — 화면 요소의 *의미*(역할·이름: "이건 검색 버튼")를 담은 트리. 원래 시각장애인 보조기술용인데, **AI 에이전트가 요소를 정확히 집는 데 매우 유용.** 깨끗한 사이트(GitHub)는 잘 나오고, 레거시는 빈약.
- **비전(Vision, 스크린샷 기반)** — 화면을 *그림으로* 보고 이해. 처음 보는·이상한 UI도 *봄*. 단 느리고·비싸고·비결정적. → *기본은 DOM, 비전은 폴백*이 우리 방침.
- **OCR / 스크린 파싱** — 스크린샷에서 글자·요소를 뽑아내는 것(OmniParser 등). 비전 보조.
- **Controlled Input(제어 입력)** — React/Vue가 값을 *내부 상태로* 관리하는 입력칸. `input.value="..."`만 넣으면 화면엔 보여도 페이지 내부 모델은 안 바뀜 → 제출 시 빈 값/실패. **native setter + 정확한 이벤트 디스패치**가 필요. *v0.4 폼 채우기의 핵심 난관.*

## 3. 에이전트 · 오케스트레이션

- **에이전트(Agent)** — 스스로 관찰→판단→행동을 반복해 목표를 수행하는 AI.
- **관찰-행동 루프(Observe–Act Loop)** — 화면 보고 → 한 행동 하고 → 다시 보고 → 반복. 클로드 인 크롬·김반장 v0.4의 기본 골격.
- **툴-콜링(Tool Calling / Function Calling)** — LLM이 "이 도구를 이렇게 써라"라고 호출. v0.3에서 도입(code_interpreter·web_search). 라우팅을 우리가 안 짜고 모델이 도구를 고르게 함.
- **MCP (Model Context Protocol)** — LLM에 도구를 표준 방식으로 연결하는 프로토콜. "get_record·save 같은 기능을 툴로 노출"의 표준. 엔터프라이즈 에이전트의 사실상 표준이 됨.
- **LangGraph** — 상태 있는 멀티에이전트를 짜는 프레임워크. **interrupt(사람에게 물어보려 멈춤)·checkpoint(상태 저장 후 이어가기)**가 HITL·다단계 작업에 딱. 김반장의 *미래 두뇌(AI 서비스)*.
- **Playwright** — MS의 브라우저 자동화 라이브러리. *별도 브라우저*를 띄워 조종 → **사내 로그인 세션을 못 탐**(우리 인증 제약과 안 맞음). 단 아이디어·요소추출 참고용.
- **RPA (Robotic Process Automation)** — 정해진 클릭 순서를 그대로 재생하는 전통 자동화(녹화-재생). 김반장은 RPA가 아니라 *상태를 보고 판단하는* 에이전트 지향.
- **HITL (Human-in-the-Loop)** — 사람이 중간에 확인·결정에 개입. 누락 보충·모호 해소·되돌릴 수 없는 저장 직전 확인. *메모리가 쌓이면 줄어듦.*
- **Bypass / 권한 모드** — 매 행동마다 안 묻고 자율 실행하는 모드(클로드 코드의 그것). 단 *되돌릴 수 없는 행동*엔 금액 상한·허용목록 등 더 높은 바.

## 4. 메모리 · 데이터

- **그래프 DB / 지식 그래프(Knowledge Graph)** — 노드(화면·상태)와 엣지(이동·행동)로 지식을 표현. **"앱을 그래프로"**: 화면=노드, 클릭 경로=엣지 → 네비게이션을 *경로 탐색*으로 풂. 김반장의 *탐색봇 → 그래프* 무기의 토대. (예: Neo4j)
- **Agent Memory(에이전트 메모리)** — 에이전트가 과거 경험·해소결과를 저장해 다음에 활용. "볼펜→품목코드", "이 폼→이 채우기 계획" 누적 → HITL 감소. v0.6의 핵심. (도구 예: Mem0, Zep)
- **RAG (Retrieval-Augmented Generation)** — 외부 문서·지식을 검색해 답에 근거로 쓰는 방식. v0.5 사내지식 검색의 기법.
- **임베딩 / 벡터스토어** — 텍스트를 의미 벡터로 바꿔 유사검색. RAG의 부품.

## 5. 우리 프로젝트 용어 (계약·모듈)

- **ScreenContext** — 익스텐션이 화면에서 추출한 *데이터*(tables/sections/charts)의 표준 형식. v0.2 읽기의 계약. 서버·LLM은 이 형식만 다룸.
- **recipe (`recipes.jsonc`)** — *읽기* 도메인 적응 힌트(어디를 읽고·버리고·가릴지). **읽기 전용 책임** — v0.4 이동/행동에 끌어쓰면 오염되므로 *분리*.
- **(v0.4) FormContext / ActionContext** — 화면의 *입력 컨트롤* 표준 형식(필드·타입·필수·옵션·버튼). ScreenContext의 쓰기판.
- **(v0.4) 네비게이션 맵** — 어디로·어떻게 가나(딥링크 또는 클릭 경로). recipe와 *별도 모듈*.
- **(v0.4) 화면 식별(Screen Identity)** — "지금 어느 화면인가"를 시그니처로 판정(도착 감지).
- **(v0.4) 액션 프로파일** — 폼별 필드·위젯·스텝·버릇.
- **fill-plan(채우기 계획)** — "어느 필드에 무슨 값"을 적은 *검사 가능한* 계획. 실행 전에 보여주고 확인받음 → 안전·HITL·테스트의 핵심 축(linchpin).
- **되돌릴 수 있는 / 없는 행동** — 보기·이동·검색·채우기(다시 하면 됨) vs 저장·제출·삭제(한 번 하면 끝). 게이트 정책의 기준.

## 6. 안전 · 거버넌스

- **거버넌스(Governance)** — 내부 데이터가 어디로 나가는지·누가 무엇을 했는지 통제. 마스킹·감사·허용목록.
- **마스킹 / deny** — 민감값을 가리거나(***), 아예 안 보냄. *데이터가 익스텐션(채널)을 떠나기 전* 적용.
- **Egress(이그레스)** — 내부 데이터가 외부(OpenAI/Anthropic 등)로 나가는 것. 프로젝트는 Anthropic·Azure OpenAI 사용을 *승인된 경계*로 둠 → 비전도 같은 경계 안.
- **Azure OpenAI vs Anthropic API** — 전자는 보통 *자사 테넌트* 내(데이터 거주·미학습) → 거버넌스 우호적. 후자는 공개 API → 정책 결이 다름. *민감작업을 어느 모델로 보낼지*가 설계 포인트.
- **Allowlist / Denylist** — 동작 허용 시스템·폼 목록 / 절대 누르면 안 되는 버튼(저장·삭제·승인) 목록. 능동 크롤·자동실행의 가드레일.
- **프롬프트 인젝션(Prompt Injection)** — 페이지 안의 악성/오류 문구가 에이전트를 속여 엉뚱한 행동을 시키는 공격. *페이지 텍스트는 데이터일 뿐 지시가 아니다*로 방어.
- **멱등성(Idempotency)** — 같은 요청을 여러 번 해도 결과가 한 번과 같음. 중복 등록 방지(저장 API에 idempotency key).
- **감사 로그(Audit Log)** — 누가·무엇을·언제·무슨 값으로 했는지 기록. 되돌릴 수 없는 행동의 안전망.

## 7. 기타 자주 나오는 것

- **SSE (Server-Sent Events)** — 서버가 클라이언트로 이벤트를 *흘려보내는* 단방향 스트림. v0.3 토큰/툴 진행 스트리밍에 사용.
- **code_interpreter / web_search** — OpenAI 네이티브(호스티드) 툴. 화면 데이터 분석 / 외부 실시간 검색. v0.3 도입.
- **drift(드리프트)** — 화면·API가 바뀌어 저장해둔 힌트/그래프가 *틀려지는* 현상. 재학습·차이 비교로 보정.
- **teach-by-demonstration** — 사람이 한 번 시연하면 그 절차를 학습하는 방식. 능동 크롤의 안전한 대안/보완.
